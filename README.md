# Лея — приватный трекер цикла

Некоммерческое приложение: без рекламы, без подписок, без слежки, без обязательного аккаунта. Данные — только на телефоне (зашифрованная БД) + бэкап в личное облако пользователя. Прогноз цикла — честный, адаптивный, на данных JMIR (НЕ «день 14»).

## Структура

```
lea_design/              # дизайн-система (отдельный пакет для серии)
lea/
  lib/
    main.dart            # init зашифрованной БД + ProviderScope
    app.dart             # MaterialApp, тема, гейт онбординга, навигация, FAB
    core/
      prediction/        # predictCycle — ядро (чистый Dart, тесты)
      database/          # Drift: таблицы, шифрование, сиды, ключи настроек
      providers/         # Riverpod: БД, прогноз, онбординг, тема
    features/
      onboarding/        # 3 вопроса → заводит прогноз
      calendar/          # главный экран: прогноз + сетка месяцев с фазами
        widgets/         # ячейка дня (фазы/прогноз пунктиром), сетка месяца
        day_phase.dart   # логика раскраски дня
      day_entry/         # ввод дня (категории из справочника)
      stats/ settings/   # статистика, настройки
  test/
    prediction/          # тесты алгоритма
    calendar/            # тесты раскраски фаз
```

## Запуск

```bash
cd lea_design && flutter pub get && cd ../lea
flutter pub get
dart run build_runner build --delete-conflicting-outputs   # генерит app_database.g.dart
flutter test          # тесты ядра + календаря
flutter run
```

> `app_database.g.dart` генерируется build_runner'ом (нет в архиве — это норма Drift).

## Готово
- Дизайн-система (4 темы, все токены).
- Ядро прогноза + тесты.
- БД (Drift) + шифрование (SQLCipher) + ключ в Keystore + гибкий справочник категорий.
- Онбординг (3 вопроса на данных) → прогноз заводится сразу.
- Главный экран: карточка прогноза + сетка 3 месяцев с фазовой заливкой и размытым прогнозом (пунктир), выбор дня → отметить/убрать менструацию, легенда.
- - Защита входа: PIN (хэш+соль в Keystore) + биометрия, экран-замок при старте.
- Бэкап: экспорт всех данных в зашифрованный ПАРОЛЕМ файл (.leabak) через системный share; восстановление через file_picker. Сервер не участвует.
- - Статистика: графики длины циклов (CustomPaint), средний/мин/макс, оценка регулярности.
- Уведомления: локальные напоминания (период/овуляция), гранулярные галочки, НЕЙТРАЛЬНЫЙ текст («Событие через N дней») для приватности.
- - Бэкап Яндекс.Диск: загрузка/скачивание зашифрованной копии в личную папку приложения (REST API). Нужен свой client_id (oauth.yandex.ru) — указан как TODO в YandexAuth.
- Реактивность: отметил день → пересчитался прогноз → перекрасился календарь.

## Дальше (не реализовано)

- OAuth-кнопка Яндекса в UI (клиент готов).

- Rive-иконки вместо материаловских, нейтральные уведомления, полноценный .arb.

## Принципы
- Прогноз не хранится — на лету из дат.
- Категории — данные (скрывать/добавлять), не хардкод.
- Бэкап — экспорт под паролем пользователя, не копия sqlite.
- Нет сервера → нет 152-ФЗ.


## Android-сборка

Обвязка в `lea/android/`: манифест с правами (уведомления, биометрия), MainActivity (ru.lea.app), gradle (minSdk 23 под SQLCipher, desugaring под уведомления), нейтральное имя «Лея» + адаптивная иконка.

Перед первой сборкой:
1. `flutter create . --platforms=android` в папке lea — НЕ нужно, обвязка уже есть; но если Flutter попросит — он дополнит недостающее, не перезаписывая наше.
2. Растровые иконки для Android <8: сгенерировать (Image Asset в Android Studio или flutter_launcher_icons). Векторная адаптивная иконка для 8+ уже есть.
3. `local.properties` создаётся автоматически при `flutter run` (путь к SDK).

Права в манифесте: POST_NOTIFICATIONS, SCHEDULE_EXACT_ALARM, USE_BIOMETRIC, boot-receiver для отложенных уведомлений. allowBackup=false (данные не утекают в авто-бэкап Google).


### Настройка Яндекс.Диска (опционально)
1. oauth.yandex.ru → приложение «для авторизации пользователей», право `cloud_api:disk.app_folder`.
2. Redirect URI: `https://oauth.yandex.ru/verification_code`.
3. Вписать client_id в `lib/core/backup/yandex_auth.dart`.
Флоу: кнопка «Сохранить на Яндекс.Диск» открывает браузер (url_launcher) → пользователь жмёт «Разрешить» → копирует показанный токен → вставляет в приложение. client_secret НЕ нужен (implicit grant). Токен хранится в Keystore.
